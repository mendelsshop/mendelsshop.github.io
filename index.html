  • git function history

    About:

    Have you ever wanted to see how a given function changes over time? That is what git_function_history is for.
    Uses Tree Sitter to make it easier to add support for as many languages as possible.

    Usage:

    It allows you to search in a git repository for all functions with a given name.
    You can use certain search filters to only look in certain folders or during certain dates among others.
    After your search finds results you can filter down further by syntactic properties of the function.

    You can use from the TUI or GUI frontend, and there is also a rust crate, that you can integrate into your own projects.

  • everything

    About:

    A joke Programming Language that combines lots of feature from many different languages into one.
    Initially based on UMPL.
    It has racket like syntax, but you can also write c/rust syntax.
    It has a module and macro system similar to racket's.
    Functions are auto curried an idea taken from ML/Haskell.
  • strings

    About:

    A ML based programming language.
    Started of as a way to learn about type inference.
    Type system is HM with row types, eventually going to use a full constraint based approach.
    Has interesting syntax where normally whatever you type is just interpreted as something to print.
    To actually program you have to quote the program (inverse of most languages).
    This idea and using ML based syntax made it it bit complicated to parse.

    About:
